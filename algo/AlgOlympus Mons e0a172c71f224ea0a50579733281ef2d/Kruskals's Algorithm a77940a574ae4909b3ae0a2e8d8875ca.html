Kruskals's Algorithm

Kruskal's approach isolates all of the nodes in the original graph, forming a forest of single node trees, and then progressively merges these trees, joining any two of all the trees with some edge of the original network at each iteration. All edges are sorted by weight before the algorithm is run (in non-decreasing order). The unification procedure then begins: choose all edges from first to last (in sorted order), and if the endpoints of the presently selected edge belong to distinct subtrees, these subtrees are united, and the edge is added to the answer. After iterating through all of the edges, we'll find that all of the vertices belong to the same sub-tree, and we'll have the MST of the given graph.

Why does greedy work here?

Why Kruskal's algorithm works is an excellent question. Why does the greedy method guarantee that an MST tree will be found? Let's look at what occurs if the graph's minimal weight edge isn't included in the spanning tree. Consider the case when the minimum weight edge 5↔65\leftrightarrow65↔6﻿ is missing from the prior graph's spanning tree. We don't know the actual structure of a spanning tree like this, but it must have some edges. Assume that the tree looks like this:

The above tree, on the other hand, cannot be an MST for the graph. This is because we can remove one of the tree's edges and replace it with the minimal weight edge 5↔65\leftrightarrow65↔6﻿. As a result, the weight of the spanning tree is reduced.

As a result, in order to build a minimum spanning tree, it is always best to include the minimum weight edge in the tree. Using similar reasoning, we can prove that adding the following edge in weight order to the tree is similarly optimum, and so on. As a result, Kruskal's approach is valid, and the MST is always produced.

Implementation

struct Edge {
    int u, v, weight;
    bool operator<(Edge const& other) {
        return weight < other.weight;
    }
};

int n;
vector<Edge> edges;

int cost = 0;
vector<int> tree_id(n);
vector<Edge> result;
for (int i = 0; i < n; i++)
    tree_id[i] = i;

sort(edges.begin(), edges.end());

for (Edge e : edges) {
    if (tree_id[e.u] != tree_id[e.v]) {
        cost += e.weight;
        result.push_back(e);

        int old_id = tree_id[e.u], new_id = tree_id[e.v];
        for (int i = 0; i < n; i++) {
            if (tree_id[i] == old_id)
                tree_id[i] = new_id;
        }
    }
}
