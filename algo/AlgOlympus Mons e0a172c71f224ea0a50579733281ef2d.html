The various zones and cities of AlgOlympus Mons are more connected to each other than ever before and new paths are under construction already. But we need to prioritize the construction of some paths before the others since the philosophy behind AlgOlympus Mons is to do every task as efficiently as possible.

But how do we prioritize which paths to construct first?

One way is to look at the redundancy of these paths. Sandstorms are very common on mars and they can create dunes that can sometimes even block some paths. We need to prioritize the construction of the paths in such a way that even if one of these paths is blocked, we have other routes to our destination.

For this, we find the bridges of this graph, that is, all those edges or paths, that if removed can make our colony disconnected.

Chapter 4: Highway

Our mars base is coming along really well now. So the newly formed government of Mars has decided to convert some of the existing paths to a highway connecting all the major points in the colony. The Highway needs to connect all the main facilities of the base, but it shouldn't be extremely expensive to construct. Therefore we want the length of the highway to be as little as possible. Hence we need the Minimum spanning tree of the graph for our base.

Let our graph GGG﻿ have nnn﻿ vertices and mmm﻿ edges. We wish to find the least weighted or minimum spanning tree for this network that connects all of the vertices (i.e. the sum of weights of edges is minimal). A spanning tree is a set of edges in which every vertex may be reached by precisely one simple path from any other. A minimal spanning tree is defined as the spanning tree with the least weight.

MSTs have the following properties:

  • If the weights of all the edges are distinct, a graph's MST is unique. Otherwise, numerous minimal spanning trees may exist. Typically, certain algorithms produce one of the possible MSTs.
  • By changing the signs of the weights of all the edges to their opposites and then performing any of the minimum spanning tree algorithms, the maximum spanning tree (spanning tree with the largest sum of weights of edges) of a graph may be created in the same way as the minimum spanning tree.

Ford Fulkerson Algorithm for Maximum flow

In a graph, the Ford–Fulkerson method determines the greatest of or maximum flow from the source or the sink. The method starts with a zero-flow flow and creates a path from the source to the sink that generates increasing flow at each step. The maximum flow has been found when the algorithm can no longer increase the flow. The technique makes use of a specific graph representation in which each original edge has a reverse edge in the opposite direction. Each edge's weight indicates how much more flow might be routed via it. The weight of each original edge matches the edge's capacity at the start of the algorithm, and the weight of each reverse edge is zero.

The concept is that increased flow reduces the quantity of future flow that can pass through the edges. On the other hand, if it turns out that routing the flow in a different direction would be more advantageous, flow can be canceled later using the graph's reverse edges. Two more rotations are required to obtain the maximum flow. For example, we can pick between the pathways 1 →2→ 3→ 6 and 1→ 4→5→ 3 → 6. Both approaches increase the flow by one.

There is no way to enhance the flow since there is no channel with positive edge weights from the source to the sink. As a result, the algorithm ends, and the maximum flow is 7.

Pseudocode

* Set flow_total = 0
* Repeat until there is no path from s to t:
   * Run Depth First Search from source vertex s to find a flow path to end vertex t
   * Let f be the minimum capacity value on the path
   * Add f to flow_total
* For each edge u → v on the path:
   * Decrease capacity of the edge c(u → v) by f
   * Increase capacity of the edge c(v → u) by f

Finding Paths

The Ford–Fulkerson method does not indicate how we should select the flow-increasing pathways. In either situation, the process will eventually end and accurately locate the maximum flow. However, the algorithm's efficiency is determined on how the pathways are chosen. Depth-first search is a straightforward approach to locate pathways. This usually works, but in the worst-case scenario, each path only improves the flow by one, and the process is sluggish. Fortunately, we may prevent this scenario by employing one of the solutions listed below:

  • Each path is chosen via the Edmonds–Karp algorithm such that the number of edges on the path is as little as feasible. This may be accomplished by searching for pathways using a breadth-first approach rather than a depth-first approach. It can be demonstrated that this ensures that the flow rises rapidly, and that the algorithm's time complexity is O(m2n)O(m^2n)O(m2n)﻿.
  • To locate pathways where each edge weight is at least a threshold value, the scaling algorithm employs depth-first search. Initially, the threshold value is a high number, such as the sum of the graph's edge weights. When no path can be discovered, the threshold value is always divided by two. The algorithm has a temporal complexity of O(m2log⁡c)O(m^2 \log{c})O(m2logc)﻿, where c is the initial threshold value. In reality, the scaling method is simpler to implement since pathways may be found via depth-first search.
