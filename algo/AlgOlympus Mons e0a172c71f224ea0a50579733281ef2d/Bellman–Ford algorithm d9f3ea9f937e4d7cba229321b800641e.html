The Bellman-Ford algorithm calculates the shortest pathways between any two beginning nodes in a network. The technique can handle any graph as long as it doesn't contain a cycle with a negative length though it can detect a negative cycle in the graph. The method keeps track of the distances between the initial node and all of the graph's nodes. The distance between the beginning node and all other nodes is initially 0 and limitless. The method minimizes distances by discovering edges that shorten pathways until no more reduction is achievable.

If the graph contains a negative cycle, we may shorten any path that contains the cycle an endless number of times by repeating the cycle. As a result, the idea of the shortest path is irrelevant in this scenario. The Bellman-Ford method may be used to detect a negative cycle by running it for n rounds. The graph has a negative cycle if the final round lowers any distance. This approach may be used to find a negative cycle anywhere in the network, regardless of where the initial node is.
