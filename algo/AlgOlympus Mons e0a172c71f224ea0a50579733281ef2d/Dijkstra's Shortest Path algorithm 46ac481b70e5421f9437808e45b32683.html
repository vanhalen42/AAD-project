Dijkstra's Shortest Path algorithm

Like the Bellman-Ford method, Dijkstra's algorithm discovers the shortest routes from the initial node to all nodes in the network. Dijkstra's method has the advantage of being more efficient and capable of analyzing big networks. However, the technique requires that no edges in the network have a negative weight. Dijkstra's approach, like the Bellman-Ford algorithm, preserves and minimizes distances to nodes during the search. Dijkstra's approach is efficient because it uses the fact that there are no negative edges to process each edge in the graph just once.

Just like the Bellman-Ford algorithm, we mark the distance of the starting node as 0 and all other nodes as ∞\infty∞﻿. When a node is picked, the method traverses all edges that originate at the node and uses them to minimize distances.

Dijkstra's algorithm is a greedy algorithm, as it looks at all the edges and moves to the local minimum among them and proceeded from there.

Negative edges

The effectiveness of Dijkstra's method is due to the absence of negative edges in the graph. The method may produce inaccurate results if there is a negative edge.

Time Complexity

Because the method travels over all nodes of the graph and adds at most one distance to the priority queue for each edge, the time complexity of the naive method is O(n2)O(n^2)O(n2)﻿ but with a min priority queue, (which reduces the overall time to search the minimum edge from a vertex), the time complexity of the given implementation is O(n+mlog⁡n)O(n+ m\log{n})O(n+mlogn)﻿, where mmm﻿ is the number of edges and nn n﻿ is the number of vertices.

Implementation (Hackerearth)

#define SIZE 100000 + 1

vector < pair < int , int > > v [SIZE];   // each vertex has all the connected vertices with the edges weights
int dist [SIZE];
bool vis [SIZE];

void dijkstra(){
                                                // set the vertices distances as infinity
    memset(vis, false , sizeof vis);            // set all vertex as unvisited
    dist[1] = 0;
    multiset < pair < int , int > > s;          // multiset do the job as a min-priority queue

    s.insert({0 , 1});                          // insert the source node with distance = 0

    while(!s.empty()){

        pair <int , int> p = *s.begin();        // pop the vertex with the minimum distance
        s.erase(s.begin());

        int x = p.s; int wei = p.f;
        if( vis[x] ) continue;                  // check if the popped vertex is visited before
         vis[x] = true;

        for(int i = 0; i < v[x].size(); i++){
            int e = v[x][i].f; int w = v[x][i].s;
            if(dist[x] + w < dist[e]  ){            // check if the next vertex distance could be minimized
                dist[e] = dist[x] + w;
                s.insert({dist[e],  e} );           // insert the next vertex with the updated distance
            }
        }
    }
}
