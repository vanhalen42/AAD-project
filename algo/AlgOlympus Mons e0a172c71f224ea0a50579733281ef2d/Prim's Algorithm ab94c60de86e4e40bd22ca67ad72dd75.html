Prim's Algorithm

Prim's approach is a different way of calculating the minimal spanning tree. The procedure starts by adding a random node to the tree. Following that, the algorithm always selects the lowest-weight edge that adds a new node to the tree. Finally, all nodes in the tree have been inserted, and a minimum spanning tree has been discovered. Prim's algorithm is similar to Dijkstra's. Prim's approach merely picks the smallest weight edge that adds a new node to the tree, but Dijkstra's algorithm always selects an edge with minimal distance from the beginning node.

Algorithm

The minimum spanning tree is progressively constructed by adding edges one by one. In the beginning, the spanning tree is made up of simply one vertex (chosen arbitrarily). The spanning tree is then picked, and the minimal weight edge emanating from this vertex is inserted. Following that, the spanning tree already has two vertices. Select and add the edge with the lowest weight that has one end in an already selected vertex (i.e., a vertex in the spanning tree) and the other end in an unselected vertex. And so on, selecting and adding the edge with the smallest weight that connects one picked vertex to one unselected vertex each time. The operation is continued until the spanning tree contains all vertices (or, in other words, there are n−1n-1n−1﻿ edges).

Why does Prim's algorithm work?

Let GG G﻿ be a connected graph (which means an MST has to exist). The resultant graph discovered using Prim's approach is denoted by TTT﻿, while the least spanning tree is denoted by SSS﻿. TTT﻿ is, without a doubt, a spanning tree and a subgraph of GGG﻿. All we have to do now is establish that the weights of SSS﻿ and TTT﻿ are the same. Consider the first time we add an edge to TTT﻿ that isn't part of SSS﻿ in the algorithm. Let's call this edge eee﻿, its endpoints aaa﻿ and bbb﻿, and the collection of previously picked vertices VVV﻿ (a∈Va\in Va∈V﻿ and b∉Vb\notin Vb∈/V﻿, or vice versa).

The vertices aaa﻿ and b of the MST S are connected by a route P. We can identify an edge f on this route where one end of f is in V and the other is not.

Because the algorithm picked eee﻿ over fff﻿, the weight of fff﻿ is larger than or equal to the weight of eee﻿. The edge ee e﻿ is added to the minimal spanning tree SSS﻿, while the edge fff﻿ is removed. We formed a cycle by adding eee﻿, and since fff﻿ was also a member of the only cycle, removing it leaves the graph cycle-free. The resultant graph is still linked because we just eliminated one edge from a cycle.

Because the weight of e was not more than the weight of fff﻿, the ensuing spanning tree cannot have a bigger total weight, and it also cannot have a lesser weight because S was a minimal spanning tree. This suggests that we constructed a different minimal spanning tree by swapping the edge f with e. Moreover, e must be the same weight as f.
